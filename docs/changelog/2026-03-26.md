## 1. Team-fit Explorer Reached a Clear v1 Flow

### What Changed

- Built out the main Team-fit path end to end: guest preview, logged-in profile saving, approval-gated recommendations, and the three buckets `similar`, `complementary`, and `unexpected`.
- Split the intake into a clearer two-step structure: Step 1 for core matching fields, Step 2 for optional signals like one-line intro, MBTI, and SDGs.
- Added save-time rules so optional data stays intentional: MBTI must be all five axes or empty, and SDGs must be exactly four or empty.
- Added a concise `How it works` modal and aligned frontend copy with the current access model.

### Why It Mattered

- The feature now reads like a real product surface instead of a backend demo.
- The access ladder is clearer: preview first, save after login, recommendations after approval.
- Optional signals now enrich matching without producing half-filled or misleading profile data.

## 3. Backend and Local Startup Became Safer

### What Changed

- Made pgvector/bootstrap DDL fail soft on lock timeouts, added schema-presence detection, and skipped unavailable Postgres writes safely.
- Switched demo seeding to local embeddings during boot and kept existing demo profiles from being clobbered on every startup.
- Enriched demo profiles with full MBTI axis data, four SDGs, and Korean one-line intros, plus a targeted backfill for older local demo rows.
- Added or expanded tests for auth gating, profile rules, recommendation payloads, startup safety, and seed backfill behavior.

### Why It Mattered

- `pnpm dev` is more reliable during real local iteration.
- Local demos now actually exercise the richer Team-fit UI instead of sparse placeholder data.
- Startup no longer depends on perfect pgvector state or external embedding calls to feel usable.
