## 2. Environment and Docker Build Stabilization
Representative commits:
- `a3b4a98 chore: standardize local and docker workflows`

### What Changed
- Changed the app local env to only track `apps/*/.env.example`; actual `.env` files are now auto-generated before running.
- Consolidated the Compose demo env into a single root [compose.env](../../compose.env).
- Added frontend / backend Dockerfiles and set up a clean BuildKit-based build path.
- Separated paths so Docker and local dev do not share `node_modules`, `.venv`, or Vite cache.
- Restructured the frontend Dockerfile around `package.json` so the dependency install layer cache is reused.

### Why It Mattered
- Without this step, permission conflicts and cache corruption repeat whenever local and Docker are used alternately.
- Env roles must be clearly defined to explain "which value comes from where".

## 5. Capture Data Model Hardening
Representative commits:
- `8e3da73 refactor: link capture jobs to users`

### What Changed
- Removed the `capture_jobs.owner_user_id` string link and replaced it with an `owner_id -> users.id` FK.
- Added `User.capture_jobs` and `CaptureJob.owner` relationships.
- The API response continues to expose `owner_user_id` to avoid breaking the existing frontend contract.
- Reinforced FK enforcement in the SQLite test DB so it is actually applied.

### Why It Mattered
- The previous structure separated data per user but the DB did not guarantee referential integrity.
- FK-based cleanup was needed first so that future domains like `voice_assets`, `image_assets`, and `generation_results` can be extended the same way.
