## 2. `pnpm install` Moved Inside setup-dev.mjs

### What Changed
- `pnpm install` call added directly inside `setup-dev.mjs` (after `ensure-dev-env`, before `uv sync`)
- Removed the `&& pnpm install` suffix that was appended to the `package.json` `setup` script

### Why It Mattered
- pnpm skips or warns when `node_modules` is missing and the shell `&&` chain is not reliable in that state.
- Running `pnpm install` via `spawnSync` inside the Node script is unconditional and always executes.

## 7. Dual-Mode `DATABASE_URL` (pnpm dev + docker compose up)

### What Changed
- `apps/backend/.env` and `apps/ai-worker/.env` keep `DATABASE_URL` pointing to `localhost` (for `pnpm dev`)
- `docker-compose.yml` backend and ai-worker services add `DATABASE_URL: postgresql+psycopg://persona:persona@db:5432/persona_mirror` in the `environment:` block, which takes precedence over `env_file`

### Why It Mattered
- `pnpm dev` runs services on the host where `db` does not resolve; Docker containers need `db` (Compose service hostname).
- Without the override, switching between the two run modes required manually editing `.env` files every time.
