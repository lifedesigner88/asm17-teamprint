## 5. Team-fit Became An Interview-First Profile Flow

### What Changed

- Replaced the old structured Step 1 / Step 2 chips with a smaller intake: one problem sentence, MBTI 5 axes, SDGs, and an 800-character Markdown narrative.
- Reworked SDG selection into a centered modal picker with four square slots, then tuned the outer preview cards back up to about 80% of the original, stacked the grid to 1 column on mobile, and replaced the step-level instruction hint with a short SDGs definition.
- Simplified the Step 1 header so only the step badge and readiness state remain, removing the extra title and explanatory copy.
- Removed the extra form-intro sentence under the interview title so the page gets into Step 1 more directly.
- Widened the guest/member status panel under the interview title into a single-column full-width block instead of a narrow side card.
- Removed the interview section title itself so the entry card starts even more quietly with just the profile pill and status block.
- Made the Step 2 card itself clickable while locked, so it now opens a small guidance modal if Step 1 is still incomplete instead of feeling unresponsive.
- Removed the helper hint under the problem-statement field so Step 1 starts with a cleaner single input label and placeholder only.
- Strengthened the problem-statement and SDGs field labels so the two required Step 1 anchors stand out more clearly.
- Slightly increased the text size on the 17-card SDG picker labels so goal names like `빈곤 종식` are easier to scan in the modal.
- Removed the outer SDG summary title line so the preview block now starts directly with the selection count and cards.
- Removed the active-profile count line from the guest/member status card so that block focuses only on the preview and save-state message.
- Removed the separate `Go to step 2` button from Step 1 because the Step 2 card itself now handles the transition and lock guidance.
- Shifted the selected MBTI axis buttons from a white fill to a faint pink highlight so the dominant side reads more clearly without feeling too heavy.
- Moved the `MBTI 5 axes` title out of the purple card so it now lines up with the problem and SDGs section titles, while keeping a lightweight 16Personalities text link beside it.
- Removed the extra MBTI helper sentence under the preview badges so the card stays visually lighter.
- Updated the SDG dialog wording and widened the 17-goal picker grid to 5 columns on large screens so the modal needs less vertical scrolling.
- Increased the text size inside the `Selected goals` summary slots so the chosen SDG labels and empty-slot labels are easier to read.
- Updated the problem-statement label so Step 1 now explicitly frames the question around the user's SoMa project.
- Replaced the problem-statement example with a SoMa-project-oriented example about helping people design their own lives and growth.
- Replaced the Step 2 Markdown placeholder with section headers only, and prefilled the textarea with that same scaffold by default.
- Renamed the Step 2 heading to `800자 프롬프트` and removed the extra explanatory sentence beneath it.
- Moved the Step 2 character counter badge to the top-right corner of the card so it reads more like a fixed editor status tag.
- Removed the extra helper hint under the Step 2 textarea so the card focuses on the title, counter, and prefilled scaffold only.
- Removed the extra Step 2 field label so the textarea now sits directly under the section heading without repeating itself.
- Replaced the old `Back to step 1` button in Step 2 with a `Reset prompt` action that restores the default scaffold text.
- Shortened the Step 1 problem statement to an 80-character single-sentence limit in both the input and the explorer API contract so the field stays true to its role.
- Simplified the landing hero into a single headline, then updated that headline to `팀빌딩을 코테처럼 풀어보자!` and added a right-aligned `추천 알고리즘` button that reopens the centered recommendation-logic modal.
- Rewrote the recommendation-logic modal copy around the current v2 interview signals so it no longer describes the older chip-based profile model.
- Removed the rest of the landing explanation cards, scope/goal/guide summary panel, and collaboration CTA card so the page now moves straight from the headline into the interview flow.
- Added a new backend explorer profile model plus interview-turn history so the initial profile is saved only after 3 AI-generated follow-up questions are answered.
- Added follow-up endpoints so a saved profile can keep growing through one extra AI question at a time.
- Rebuilt the frontend around the new flow: save now opens a chat-style dialog, the saved state becomes read-only, and the recommendation area is temporarily replaced with a `coming soon` panel.
- Rewrote the team-fit locale copy and backend tests to match the new interview contract.

### Why It Mattered

- The product now behaves more like a guided team-fit interview than a static checklist.
- Raw motivation and collaboration context are now captured as conversational history instead of only rigid tags.
- The SDG step now feels more intentional and visual, while backend and frontend validation both agree on `exactly 4 required`.
- Recommendation UI is intentionally paused so the next version can be rebuilt around the new saved interview data model instead of legacy assumptions.
