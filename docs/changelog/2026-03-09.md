## 6. Dev Runtime Stabilization and Security Hardening
Representative commits:
- `0d21c49 feat: harden auth and dev template`
- `017cabc fix: polish auth flow and workspace`

### What Changed
- Reinforced the dev startup order so the backend waits for the DB to be ready before starting.
- Adjusted the frontend to also start after the backend is ready.
- Reduced conflicts so Docker and local dev do not share the same `.venv`, `node_modules`, or Vite cache.
- Resolved bcrypt conflicts, JWT secret length issues, VS Code Python warnings, and logout response errors.
- Separated the admin menu to only be visible during admin sessions.

### Why It Mattered
- In real development, "does it run reliably and repeatedly" matters more than "does the feature exist".
- Cleaning up dev/Docker conflicts, auth storage, and startup order at this stage is what keeps feature implementation speed up later.
