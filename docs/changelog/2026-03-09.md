## 6. Dev Runtime Stabilization and Security Hardening
대표 커밋:
- `0d21c49 feat: harden auth and dev template`
- `017cabc fix: polish auth flow and workspace`

### What Changed
- DB 준비 상태를 기다린 뒤 backend가 뜨도록 dev 실행 순서를 보강했다.
- frontend도 backend 준비 후 시작하도록 조정했다.
- Docker와 로컬이 같은 `.venv`, `node_modules`, Vite cache를 공유하지 않도록 충돌을 줄였다.
- bcrypt 충돌, JWT secret 길이, VS Code Python 경고, 로그아웃 응답 오류를 정리했다.
- 관리자 메뉴는 관리자 세션일 때만 보이도록 UI와 권한 흐름을 분리했다.

### Why It Mattered
- 실제 개발은 "기능이 있느냐"보다 "계속 안정적으로 반복 실행되느냐"가 더 중요하다.
- 이 단계에서 dev/Docker 충돌, auth 저장 방식, 시작 순서 같은 문제를 먼저 정리해두어야 이후 기능 구현 속도가 유지된다.

## 8. Backend Capture API and Domain Alignment

- `1e595ea refactor: align frontend and backend domains`

### What Changed
- backend에 `capture_jobs` 저장 구조를 추가했다.
- `POST /capture/jobs`, `GET /capture/jobs`, `GET /capture/jobs/{job_id}`를 도입했다.
- backend를 `app/common` + `app/features/<domain>` 구조로 재정렬했다.

- FastAPI startup 이벤트는 `lifespan` 기반으로 전환했다.

### Why It Mattered
- 이 시점부터 frontend와 backend는 둘 다 `auth`, `admin`, `capture`라는 같은 도메인 이름으로 찾을 수 있게 됐다.
- 즉 "코드가 실행된다" 수준을 넘어서, 이제는 "확장 가능한 구조"가 된 것이다.
