## 7. Step 1 — Email Signup with OTP Verification and Reset PIN

Representative commit: `1902ce2 feat: email signup with OTP verification, 4-digit PIN, and reset PIN flow`

### What Changed
- **User ID**: auto-generated 6-char alphanumeric ID (client-side shuffle + copy UI)
- **Email**: required, validated with `EmailStr`, unique in DB, used as login identifier
- **Password → 4-digit PIN**: signup and login accept only a 4-digit numeric PIN; admin login remains flexible
- **OTP via Resend**: 6-digit code emailed on signup; account cannot log in until verified (10-min expiry)
- **Reset PIN flow**: `POST /auth/reset-pin/request` → `POST /auth/reset-pin/confirm`
- **Frontend**: 3-step signup (form → OTP entry → verified card → auto-redirect to login); "Forgot PIN?" on login page
- **DB**: added `is_verified`, `otp_code`, `otp_expires_at` to `users` table
- **Admin seed**: marked `is_verified=True` automatically

### Why It Mattered
- Original signup had no email, no verification, no account recovery.

## 9. AI Chat Interview + ai-worker LangGraph Pipeline

Representative commit: `06beae9 feat: pivot interview to AI chat + add ai-worker LangGraph pipeline`

### What Changed

#### Backend — Chat interview endpoint
- `POST /capture/interview/chat`: stateless, receives full message history, returns `{ message, is_complete }`
- System prompt instructs Claude to ask 5 questions one at a time, append `[INTERVIEW_COMPLETE]` after final answer
- `CaptureInterviewPayload` changed from 4 text fields to `{ messages: ChatMessage[], isComplete: bool }`

#### Backend — CaptureJob result fields
- `result` (JSON) and `persona_id` (String(16), unique) added to `CaptureJob` model

#### ai-worker — LangGraph pipeline
- `graph.py`: single-node workflow — reads interview messages, calls Claude, returns `{ archetype, top3_values, one_liner }`
- `main.py`: polling loop (every 5s) — picks `status=pending` jobs, runs graph, writes result, sets `status=done`/`failed`

#### Frontend — Chat UI
- `InterviewCapturePage`: auto-starts on mount, user types answers, full history sent each turn, auto-navigates to review on completion
- Voice/Image pages replaced with "coming soon" placeholders

### Why It Mattered
- Static 4-field interview gave Claude nothing to work with — conversational flow produces richer persona data.
