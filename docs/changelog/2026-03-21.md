## 8. Step 1 — Email Signup with OTP Verification and Reset PIN

Representative commit: `1902ce2 feat: email signup with OTP verification, 4-digit PIN, and reset PIN flow`

### What Changed
- **User ID**: auto-generated 6-char alphanumeric ID (client-side, shuffle + copy UI before submission)
- **Email**: required, validated with `EmailStr` (`email-validator` package), unique in DB, used as login identifier
- **Password → 4-digit PIN**: signup and login accept only a 4-digit numeric PIN; admin login remains flexible
- **OTP email verification via Resend**: on signup, a 6-digit code is emailed; account cannot log in until verified (10-min expiry)
- **Reset PIN flow**: `POST /auth/reset-pin/request` sends OTP → `POST /auth/reset-pin/confirm` sets new PIN
- **Frontend**: 3-step signup (form → OTP entry → verified card → auto-redirect to login); "Forgot PIN?" link on login page; show/hide PIN toggle on both pages
- **DB**: added `is_verified`, `otp_code`, `otp_expires_at` columns to `users` table
- **Admin seed**: marked `is_verified=True` automatically; `ADMIN_SEED_EMAIL` added to env

### Why It Mattered
- The original signup had no email, no verification, and an 8-char free-text password — unsuitable even for a demo.
- OTP verification prevents throwaway account creation and establishes a real email link per user.
- The 4-digit PIN lowers the barrier for demo users while keeping the admin account secure with a longer credential.
- Reset PIN via email closes the account recovery loop without needing a separate user management UI.

## 10. AI Chat Interview + ai-worker LangGraph Pipeline (feat/ai-worker)

Representative commit: `06beae9 feat: pivot interview to AI chat + add ai-worker LangGraph pipeline`

### What Changed

#### Backend — Chat interview endpoint
- New `POST /capture/interview/chat`: stateless endpoint that receives the full message history and returns the next Claude response
- System prompt instructs Claude to ask exactly 5 questions one at a time, then append `[INTERVIEW_COMPLETE]` after the final answer
- Backend strips the signal and returns `{ message, is_complete }` to the frontend
- `CaptureInterviewPayload` changed from `{selfSummary, coreValues, speakingStyle, keywords}` to `{ messages: ChatMessage[], isComplete: bool }`
- Added `anthropic` package to backend dependencies

#### Backend — CaptureJob result fields
- `result: Mapped[dict | None]` (JSON) and `persona_id: Mapped[str | None]` (String(16), unique) added to `CaptureJob` model
- `CaptureJobResponse` now includes `result: PersonaResultResponse | None`
- `PersonaResultResponse`: `{ persona_id, archetype, top3_values, one_liner }`

#### ai-worker — LangGraph pipeline
- New `apps/ai-worker/worker/graph.py`: single-node LangGraph workflow
  - Reads `interview.messages` array from job payload
  - Calls Claude with extraction prompt → returns `{ archetype, top3_values, one_liner }`
- New `apps/ai-worker/worker/main.py`: polling loop (every 5s)
  - Picks `status=pending` jobs, runs graph, writes `result` + `persona_id` to DB
  - Sets `status=done` or `status=failed`

#### Frontend — Chat UI
- `InterviewCapturePage` replaced with a chat UI:
  - Auto-starts on mount (calls backend to get first question)
  - User types answers; full history sent to backend on each submission
  - Saves draft after each exchange; auto-navigates to `/capture/review` on completion
- `requestInterviewChat()` added to capture API utility
- `ChatMessage` type added to `types.ts`

#### Frontend — Voice/Image marked as coming soon
- `VoiceCapturePage` and `ImageCapturePage` replaced with simple "coming soon" placeholders
- Capture flow: Interview → Review (voice/image steps are still visible in the step map but disabled)
- `StepLink` component updated to support `disabled` prop (renders a non-link tile with "soon" badge)
- Progress bar shows 0/1 or 1/1 (interview only)
- Review page: `ready` check now only requires `completion.interview`; back button → `/capture/interview`
- Overview page and payload summary updated to reflect new interview structure

### Why It Mattered
- The static interview form (4 text fields) gave Claude nothing to work with — users would fill it mechanically without reflection.
- A conversational flow where Claude asks thoughtful, adaptive questions produces richer, more honest persona data.
- Keeping voice/image stubs in the UI signals the product direction without blocking the MVP launch.
