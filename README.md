# Park Sejong PR for ASM17

> 이 루트 도메인은 지금 `박세종 PR`로 바로 연결됩니다.
>
> 다만 그것이 개인 소개 페이지만 의미하지는 않습니다. 안쪽에는 여전히 SW마에스트로 17기(ASM17) 팀빌딩을 위한 대시보드, 인증, 운영 기능이 살아 있습니다.
>
> 현재 우선순위는 "이 서비스가 무엇인지"를 길게 설명하는 것보다, "박세종이 어떤 사람이고 왜 같이 팀하면 좋은지, 그리고 실제로 바로 만들 수 있는 사람인지"를 30초 안에 전달하는 것입니다.

영문 README는 [README_eng.md](README_eng.md)에서 볼 수 있습니다.

## 왜 만들었나요

처음 문제의식은 ASM17 팀빌딩이었습니다.

대면 팀빌딩 전에 지원자들이 면접 일정, GitHub, 자기소개 Notion 링크를 미리 공유하고, 같이 면접 본 사람들의 정보를 더 빨리 파악할 수 있으면 팀빌딩이 훨씬 쉬워질 수 있다고 봤습니다.

그래서 이 프로젝트는 원래 아래 질문에서 시작했습니다.

> 직접 만나기 전에 지원자들이 면접 일정, GitHub, 자기소개 링크를 공유하고, 같이 면접 본 사람들의 정보를 미리 볼 수 있다면 팀빌딩이 훨씬 쉬워지지 않을까?

하지만 2026-03-25 기준으로 루트 도메인의 역할은 한 단계 다시 정렬되었습니다.

- 루트 `/`는 `박세종 PR` 페이지 그 자체입니다.
- `박세종 PR`, `AI 세종`, 유튜브, 실제 결과물이 첫인상용 핵심 자산입니다.
- `서울 대시보드`, 인증, 운영 기능은 관심이 생긴 뒤 들어가는 2단계 도구입니다.

즉 지금의 제품은 `박세종 PR + ASM17 팀빌딩 툴`이 함께 있는 구조입니다.

## 지금 사용자가 할 수 있는 것

- 루트 `/` 또는 기존 `박세종 PR` 링크(`/persona/sejong`)에서 박세종이 어떤 사람인지, 왜 소마에 왔는지, 어떤 팀을 찾는지 빠르게 파악하기
- `박세종 PR`에서 역할, 문제의식, 팀 핏, 찾는 팀원상 확인하기
- `AI 세종` 데모로 실제 데이터 기반 질문 흐름 확인하기
- 유튜브 링크로 말투와 에너지, 사람의 결 먼저 보기
- 회원가입하고 로그인하기
- 이메일 인증과 17기 합격 인증 진행하기
- 면접 날짜와 시작 시간 등 면접 관련 정보 입력하기
- GitHub 링크 추가하기
- 자기소개 Notion 링크 추가하기
- 같이 면접 본 사람들의 정보 확인하기
- 팀빌딩 전에 가능한 팀원을 더 빠르게 파악하기

## 현재 핵심 사용 흐름

1. 사용자가 루트 `/`로 들어오면 곧바로 `박세종 PR` 화면 자체에서 박세종의 문제의식과 팀업 방향을 먼저 봅니다.
2. 이어서 `박세종 PR`, `AI 세종`, 유튜브, 실제 결과물 링크로 "말만 하는 사람인지 아닌지"를 빠르게 확인합니다.
3. 관심이 생기면 회원가입, 로그인, 이메일 인증, 17기 합격 인증으로 넘어갑니다.
4. 사용자는 면접 날짜와 시작 시간 등 면접 관련 정보를 입력합니다.
5. GitHub 링크와 자기소개 Notion 링크를 추가합니다.
6. 시스템이 사용자를 면접 슬롯 안에 배치합니다.
7. 사용자는 서울 대시보드에서 같이 면접 본 사람들의 정보를 보고, 필요하면 운영 기능과 체크 흐름을 사용합니다.
8. 그 정보를 바탕으로 실제로 이야기해볼 만한 사람을 더 빨리 판단합니다.

## 주요 기능

- 루트 `/` 자체가 `박세종 PR`인 사람 중심 첫 진입
- 제품 제작자의 `박세종 PR` 페이지
- 로그인 사용자 전용 `AI 세종` 멀티턴 채팅 페이지
- 이메일 회원가입, 로그인, PIN 재설정
- 17기 이메일 인증 / 합격 인증 기반 접근 제어
- SW Maestro 17 지원자를 위한 면접 일정 입력
- GitHub 프로필 링크 입력
- 자기소개 Notion 링크 입력
- 같이 면접 본 사람들을 빠르게 보는 대시보드 / 슬롯 기반 탐색
- 상세 멤버 보기 접근을 위한 관리자 승인 플로우
- 한국어 / 영어 UI 지원

## AI 세종은 어떻게 작동하나요

현재 `AI 세종`은 단순한 한 줄 챗봇이 아니라, 팀빌딩용으로 설계한 `로그인 기반 멀티턴 채팅`입니다.

- 로그인한 사용자만 사용할 수 있습니다.
- 질문과 답변은 DB에 저장됩니다.
- 답변할 때는 최근 대화 일부를 다시 넣어 멀티턴 맥락을 유지합니다.
- 세종의 persona 데이터, creator PR 데이터, 그리고 Hupository 핵심 문서를 함께 참고합니다.
- 현재는 `profile/persona/snapshot`에 더해 `decisions.yaml`과 최신 `session.md`를 기본 참고하고, `오늘/요즘/고민` 성격의 질문에는 최신 일일 저널도 조건부로 붙입니다.
- 무분별한 사용을 막고, 나중에 자주 묻는 질문 패턴을 보기 위해 기록을 남깁니다.

아직 완성형 AI 기능이라고 보기는 어렵습니다. 다만 `실제 팀빌딩 맥락에서 어떤 질문이 나오고, 어떤 정보가 더 필요해지는지`를 바로 볼 수 있다는 점에서 제품 실험 가치가 있습니다.

## 앞으로의 방향

현재 범위는 의도적으로 좁게 잡았습니다.

지금 목표는 팀빌딩 전에 사람을 더 잘 파악하게 만드는 것입니다. 아직 다듬어진 대규모 커뮤니티 플랫폼은 아닙니다.

만약 충분한 사용자가 모이고, 프로필 데이터가 어느 정도 쌓인다면 다음 단계로는 아래 방향을 생각하고 있습니다.

- 비전이나 방향성이 비슷한 사람을 추천하는 AI 보조 기능
- 더 나은 팀빌딩 후보 탐색과 shortlist 지원
- creator 페이지를 넘어서는 참가자 프로필 화면 확장
- `AI 세종`의 grounding, retrieval, 대화 UX, 질문 패턴 분석을 더 고도화

이 항목들은 로드맵 아이디어이며, 현재 완성된 기능은 아닙니다.

## 함께 고도화하고 싶은 부분

이 프로젝트는 이미 돌아가고 있지만, 아직 같이 키울 여지가 많습니다.

- AI가 어떤 데이터를 얼마나 참고해야 자연스럽고 신뢰감 있게 답하는지
- 자주 묻는 질문을 제품 개선으로 어떻게 연결할지
- 팀빌딩에 정말 도움이 되는 추천 / 필터 / 대화 유도 UX가 무엇인지

특히 `AI 기능을 더 정확하고 유용하게 만들고 싶은 사람`, `제품 관점에서 이 흐름을 더 다듬고 싶은 사람`과 함께 고도화해보고 싶습니다.

## 로컬 개발 / 실행 방법

### 준비물

- Node `24.11.0`
- Python `3.11.15`
- `pnpm`
- `uv`
- Docker 선택 사항

### 최초 설정

```bash
git clone <repo-url>
cd 260309_persona-mirror
node scripts/setup-dev.mjs
```

초기 설정 이후에는 아래 명령을 사용합니다.

```bash
pnpm setup
```

### 환경 변수

`apps/frontend/.env`, `apps/backend/.env`는 필요할 때 `.env.example`을 기반으로 생성됩니다.

백엔드와 AI 워커를 사용하려면 다음 값을 채워야 합니다.

```text
DATABASE_URL=...
ANTHROPIC_API_KEY=...
RESEND_API_KEY=...   # optional
```

AI 워커를 쓴다면 `apps/backend/.env`와 `apps/ai-worker/.env`에 같은 `DATABASE_URL`을 넣어야 합니다.

휴포지토리 기반 응답은 기본적으로 `apps/backend/hupository`를 먼저 읽습니다.
다른 위치를 쓰고 싶다면 `apps/backend/.env`에서 `HUPOSITORY_ROOT`로 경로를 지정할 수 있습니다.

### 로컬 실행

```bash
pnpm dev
```

주요 로컬 주소:

- Frontend: `http://localhost:3000`
- 메인 진입 화면 / 박세종 PR 페이지: `http://localhost:3000/`
- 기존 박세종 PR 링크: `http://localhost:3000/persona/sejong` -> `http://localhost:3000/`
- AI 세종 페이지: `http://localhost:3000/ai/sejong`
- 서울 대시보드: `http://localhost:3000/seoul/dashboard`
- Backend API 문서: `http://localhost:8000/docs`

### Docker 실행

```bash
pnpm docker
pnpm docker:logs
pnpm docker:down
```

### 품질 확인

```bash
pnpm test:backend
pnpm lint
pnpm format
```
